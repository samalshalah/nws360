You are building NWS360 (Competitive Media Monitoring System) with ONE universal UI.
All screens exist, but features are hidden dynamically based on role + permissions + plan tier.

Core principle:
- Same React layout for everyone
- Sidebar items and page actions appear ONLY if allowed
- Backend must also enforce permissions (no UI-only security)

Multi-tenant isolation is mandatory.
AI is ONLY allowed inside Analytics + Intelligence (never in feed ingestion/rendering).

========================================
1) ROLES (AUTH LEVEL)
========================================
System roles (high level):
1) GLOBAL_ADMIN
2) CLIENT_ADMIN
3) CLIENT_USER

Client user types (sub-roles under CLIENT_USER):
- READER (consumes feed only)
- ANALYST (analytics + reports)
- EDITOR (curates, tags, brief drafts)
- MONITOR (ops/health view, no analytics deep)
- EXECUTIVE (executive home + brief view only)
- INTEGRATIONS_MANAGER (manages integrations/webhooks, if plan allows)

IMPORTANT:
CLIENT_ADMIN can assign these user types to users in their tenant.
GLOBAL_ADMIN can assign anything and impersonate.

4) PLAN TIERS (FEATURE GATES)
========================================
Implement planTier gates that hide entire sections:

Starter:
- Feed + Saved + Basic Sources + Basic Keywords
- Analytics: Overview + Volume only
- Intelligence: hidden unless purchased AI add-on

Pro:
- Full analytics suite
- Collaboration
- Limited integrations (1-2)
- Intelligence available if AI add-on enabled

Enterprise:
- Everything
- Advanced integrations, SSO, audit logs, custom exports
- Executive Home, Predictive Intelligence

AI add-on is separate:
- aiEnabled must be true AND user must have INTELLIGENCE_* permissions
- If aiEnabled is false: Hide ALL Intelligence menu and ALL AI buttons.
